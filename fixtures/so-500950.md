## QBO truth at start of silo 8


Paid in QBO. Local row out of sync.

## Why it was invisible in Ship Station

`PaidUnshippedOrdersList.tsx` line 284 isPaid check:
```ts
const isPaid = o.is_paid === true || (!!o.paid_at && Number(o.balance_due || 0) === 0);
```
Both branches failed. Row appeared gray (not Ready) but was IN the queue. Easy to mistake for "row missing."

## Why catch-up never reached it

- `qbo-invoice-sync` cron checks 21 oldest open invoices per run. 63863 wasn't in the 21 oldest after ~3 weeks.
- Pre-silo-8 "Sync QBO" button in Ship Station only re-checked orders created in the last 7 days. 500950 was 44 days old.
- Gap between the two windows = silent stuck state.

## Fix applied silo 8

```sql
UPDATE public.sales_orders
SET is_paid = true, paid_at = now()
WHERE id = 'a5e827c2-1eb9-4094-b3ba-c79d020d3667';
```

Row now appears Ready (green) in Paid Unshipped.
